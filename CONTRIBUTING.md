CONTRIBUTING
============

Thank you for your interest in contributing to Orbit.

This document explains how to submit issues and pull requests, coding standards, and how to prepare your changes.

1. Filing issues
   - Search existing issues before opening a new one.
   - Provide a clear title and reproduction steps. Include environment details (OS, Rust version).

2. Branching and commits
   - Create a descriptive branch name: `feat/short-description` or `fix/short-description`.
   - Keep commits small and focused. Write clear commit messages in English.

3. Code style
   - Follow Rust's idioms and formatting. Run `cargo fmt` before committing.

4. Tests
   - Add tests for new features or bug fixes where practical. Run `cargo test` and ensure they pass.

5. Pull Requests
   - Describe the problem, the change, and testing performed. Link related issues.

If you're unsure where to start, open an issue and label it `good first issue`.

CONTRIBUYENDO
=============

Gracias por tu interés en contribuir a Orbit.

Este documento explica cómo reportar issues y enviar pull requests, estándares de código y cómo preparar tus cambios.

1. Reportar issues
   - Busca issues existentes antes de crear uno nuevo.
   - Proporciona un título claro y pasos para reproducir. Incluye detalles del entorno (SO, versión de Rust).

2. Ramas y commits
   - Usa nombres de rama descriptivos: `feat/descripcion-corta` o `fix/descripcion-corta`.
   - Mantén los commits pequeños y enfocados. Mensajes de commit claros en inglés.

3. Estilo de código
   - Sigue las convenciones de Rust. Ejecuta `cargo fmt` antes de commitear.

4. Tests
   - Añade tests para nuevas características o correcciones. Ejecuta `cargo test`.

5. Pull Requests
   - Describe el problema, el cambio y las pruebas realizadas. Relaciona issues si aplican.

Si no sabes por dónde empezar, abre un issue y márcalo con `good first issue`.
